# দাঁড়ািপাল্লা (Daripallah) - ফাইনাল ভাইভা প্রস্তুতি ও প্রজেক্ট গাইড (Viva Guide)

এই ডকুমেন্টটি আপনার ফাইনাল ভাইভা পরীক্ষার প্রস্তুতির জন্য তৈরি করা হয়েছে। এখানে প্রজেক্টের প্রযুক্তিগত বিবরণ, গিটহাব আপডেট পুশ করার নিয়ম এবং প্রজেক্ট তৈরি করার সময় ফেস করা বিভিন্ন চ্যালেঞ্জের বিস্তারিত উত্তর দেওয়া হয়েছে।

---

## ১. প্রজেক্টের পরিচিতি (Project Introduction)
দাঁড়িপাল্লা (Daripallah) হলো একটি **AI-Powered SaaS Multi-vendor Marketplace**। এর মাধ্যমে যেকোনো ক্ষুদ্র বা মাঝারি ব্যবসায়ী মাত্র ১ মিনিটে নিজস্ব অনলাইন সাবডোমেইন স্টোর খুলতে পারেন। গ্রাহকরা বাংলা এবং ইংরেজি উভয় ভাষায় পণ্য সার্চ করে খুব সহজে অর্ডার করতে পারেন। এছাড়া ড্যাশবোর্ডে "স্মার্ট ইনভেন্টরি" যুক্ত রয়েছে যা কাস্টমারদের জন্য স্বয়ংক্রিয় প্রাইজ ক্যালকুলেটর হিসেবে কাজ করে।

## ২. এই প্রজেক্টটি তৈরিতে কোন কোন ল্যাঙ্গুয়েজ ও প্রযুক্তি ব্যবহার করা হয়েছে?
প্রজেক্টটি সম্পূর্ণ আধুনিক ওয়েবের স্ট্যান্ডার্ড মেনে তৈরি করা হয়েছে:

*   **প্রোগ্রামিং ভাষা (Programming Languages):**
    *   **JavaScript (ES6+):** পুরো অ্যাপ্লিকেশনের লজিক ও ইন্টারঅ্যাকশন জাভাস্ক্রিপ্ট দিয়ে তৈরি।
    *   **HTML5 & CSS3:** সাইটের বেসিক স্ট্রাকচার ও আধুনিক ভিজ্যুয়াল স্টাইলিংয়ের জন্য ব্যবহৃত হয়েছে।
*   **ফ্রন্টেন্ড ফ্রেমওয়ার্ক (Frontend Framework):**
    *   **Next.js (App Router - React 19):** হাইব্রিড স্ট্যাটিক ও সার্ভার সাইড রেন্ডারিং (SSR/ISR) এবং অপ্টিমাইজড রাউটিংয়ের জন্য ব্যবহৃত হয়েছে।
    *   **Tailwind CSS:** সাইটের রেসপনসিভ ডিজাইন এবং দ্রুত স্টাইলিং নিশ্চিত করতে ব্যবহৃত হয়েছে।
    *   **Lucide React:** আকর্ষণীয় আইকন সিস্টেমের জন্য।
*   **ডাটাবেজ ও ব্যাকেন্ড (Database & Backend):**
    *   **Firebase & Firestore:** গ্রাহকদের তথ্য, পণ্যের বিবরণ, অর্ডার ট্র্যাকিং এবং শপ কনফিগারেশন রিয়েল-টাইমে সেভ ও সিঙ্ক করার জন্য।
    *   **Firebase Authentication:** গুগল লগইন এবং ইমেইল-পাসওয়ার্ড সুরক্ষার মাধ্যমে ওটিপি/লগইন ম্যানেজমেন্টের জন্য।
    *   **Firebase Storage / Cloudinary:** ব্র্যান্ডের লোগো, পণ্যের ছবি এবং কাস্টমারদের পেমেন্ট স্ক্রিনশট আপলোড করে সেভ করার জন্য।

## ৩. গিটহাবে প্রজেক্টের আপডেট কীভাবে পুশ করব? (Step-by-Step GitHub Guide)
গিটহাবে আপডেট পুশ করার জন্য নিচের ধাপগুলো অনুসরণ করতে হবে:

1.  **টার্মিনাল বা কমান্ড প্রম্পট খুলুন:** আপনার প্রজেক্ট ডিরেক্টরিতে (`d:\webmaa`) টার্মিনাল ওপেন করুন।
2.  **কাজের স্ট্যাটাস চেক করুন:**
    ```bash
    git status
    ```
    *(এর মাধ্যমে দেখতে পারবেন কোন কোন ফাইলে আপনি পরিবর্তন বা কোড এডিট করেছেন।)*
3.  **পরিবর্তনগুলো যুক্ত করুন (Stage Changes):**
    ```bash
    git add .
    ```
    *(সব নতুন ও পরিবর্তিত ফাইলগুলোকে কমিট করার জন্য রেডি করতে এই কমান্ড ব্যবহার করা হয়।)*
4.  **কমিট মেসেজ দিয়ে সেভ করুন (Commit Changes):**
    ```bash
    git commit -m "feat: fixed logo loading delay, common order UI, and smart inventory search"
    ```
    *(এখানে মেসেজটি সুনির্দিষ্ট হতে হবে যেন পরবর্তীতে পরিবর্তনগুলো ট্র্যাক করা যায়।)*
5.  **রিমোট রিপোজিটরিতে পুশ করুন (Push to GitHub):**
    ```bash
    git push origin main
    ```
    *(এটি আপনার লোকাল কম্পিউটারের কোডগুলোকে গিটহাবের `main` ব্রাঞ্চে আপলোড বা পুশ করে দিবে।)*

## ৪. প্রজেক্টের প্রধান ফিচারগুলো কী কী?
ভাইভাতে যদি জানতে চায় এই প্রজেক্টের স্পেশালিটি বা প্রধান ফিচার কী, তবে এই পয়েন্টগুলো বলবেন:
1.  **Instant Store Builder:** যেকোনো বিক্রেতা এক মিনিটে নিজের সাবডোমেইন (যেমন: `messerbazar.daripallah.com`) বা কাস্টম ডোমেইনে দোকান চালু করতে পারেন।
2.  **English-to-Bangla Phonetic Search:** কোনো ইউজার ইংরেজিতে "alu" বা "murgi" লিখলেও ডাটাবেজ থেকে বাংলায় থাকা "আলু" বা "মুরগি" পণ্যগুলো সার্চ রেজাল্টে শো করবে।
3.  **Smart Inventory Calculator:** পণ্য বিক্রির জন্য ওজনের অনুপাত সেটআপ (যেমন: ১ কেজি আলু = ৪০ টাকা)। কাস্টমার ৪০০ গ্রাম চাইলে সিস্টেম স্বয়ংক্রিয়ভাবে দাম নির্ধারণ করে দিবে।
4.  **Common Order Sheet:** একটি সিঙ্গেল পেইজের টেবিল গ্রিড ব্যবহার করে কাস্টমাররা দ্রুত একের অধিক পণ্য আলাদা আলাদা ওজনে এক ক্লিকে কার্টে নিয়ে অর্ডার করতে পারেন।
5.  **PWA (Progressive Web App):** কাস্টমাররা সরাসরি গুগল ক্রোম বা আইওএস সাফারি ব্রাউজার থেকে এই ওয়েবসাইটকে অ্যাপ্লিকেশন হিসেবে মোবাইলে ইন্সটল করতে পারবেন।

## ৫. এই প্রজেক্ট তৈরি করার সময় কী কী কঠিন সমস্যা বা চ্যালেঞ্জ ফেস করা হয়েছে? (Challenges & Solutions)
শিক্ষক যদি জিজ্ঞেস করেন **"কী কী সমস্যা ফেস করছ এবং কীভাবে সমাধান করছ?"** তখন নিচের উদাহরণগুলো দিবেন:

*   **১. রিয়্যাক্ট স্টেট ব্যাচিং সমস্যা (React State Batching Issue):**
    *   *সমস্যা:* যখন ইউজার "কমন অর্ডার শিট" থেকে একসাথে অনেকগুলো প্রোডাক্ট কার্টে যোগ করতে যেতেন, তখন লুপের ভেতর `addToCart` একাধিকবার কল করা হতো। কিন্তু রিয়্যাক্টের স্টেট আপডেটের আসিনক্রোনাস (Asynchronous) স্বভাবের কারণে কার্টে সর্বশেষ প্রোডাক্টটি বা ১-২টি প্রোডাক্ট বাদে বাকিগুলো যোগ হতো না।
    *   *সমাধান:* আমরা লুপের ভেতর সরাসরি স্টেট না বদলে, লোকাল মেমোরিতে একটি নতুন কার্ট অ্যারে (`updatedCart`) তৈরি করি এবং সেখানে সমস্ত প্রোডাক্ট পুশ করে লুপের বাইরে একবারে `setCart(updatedCart)` এবং লোকাল স্টোরেজ আপডেট সম্পন্ন করি। এর ফলে এখন ১০০% সঠিকভাবে সব প্রোডাক্ট কার্টে যোগ হয়।
*   **২. ফোনেটিক সার্চ ইঞ্জিন ডিজাইন (Phonetic Transliteration Engine):**
    *   *সমস্যা:* কাস্টমারদের মধ্যে অনেকে ইংরেজি হরফে বাংলা টাইপ করেন (যেমন: "goru" বা "piyaj")। সাধারণ ডেটাবেজ সার্চ দিলে এই ধরণের প্রোডাক্ট পাওয়া যেত না।
    *   *সমাধান:* আমরা একটি কাস্টম লাইব্রেরি ফাংশন `normalizePhonetic` তৈরি করি, যা ভাওয়েল বাদ দিয়ে ব্যঞ্জনবর্ণ ও কমন সাউন্ড প্যাটার্ন দিয়ে একটি ডিকশনারি ম্যাপিং চালায় (যেমন: `sh` → `s`, `z` → `j`, `c` → `k`)। এর মাধ্যমে কাস্টমার যেকোনো হরফে সার্চ দিলে সঠিক বাংলা প্রোডাক্ট খুঁজে পায়।
*   **৩. লোডিং টাইম ও লোগো মিসম্যাচ ল্যাগ (Loading Logo Lag):**
    *   *সমস্যা:* পেজ লোড হওয়ার প্রথম ২-৩ সেকেন্ড সাইটে একটি জেনেরিক দাঁড়িপাল্লা বা পুরোনো লোগো আসতো, কারণ ডাটাবেজ থেকে আসল লোগোর URL আসতে ২ সেকেন্ড সময় নিত।
    *   *সমাধান:* আমরা লোকাল `public` ডিরেক্টরির সমস্ত স্ট্যাটিক ফাইলে (`logo.png`, `favicon.ico`) আসল লোগোটি ডাউনলোড করে ওভাররাইট করে দেই। এছাড়াও Next.js এর `loading.js`-কে ক্লায়েন্ট কম্পোনেন্ট বানিয়ে `useParams` দিয়ে শপ-স্ল্যাগ বের করে `localStorage` থেকে পূর্বে ক্যাশ করা লোগো সরাসরি প্রথম মিলিসেকেন্ডেই শো করি। ফলে কোনো ল্যাগ বা লোগো জাম্প ছাড়াই স্মুথলি পেজ লোড হয়।
